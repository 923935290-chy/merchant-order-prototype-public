# 示例商城类目管理 PRD

## 一、需求背景

示例商城目前已在 C 端商品流中引入类目展示，用于承接“数码、穿搭、养护、食品、车内、气味”等业务一级入口，以及一级类目下的二级图片入口。现阶段类目配置缺少统一后台，运营无法独立维护类目结构、展示顺序、启停状态和二级类目图片，类目调整依赖研发或临时配置，效率低且容易出错。

从业务上看，类目需要同时满足两类诉求：

- C 端导购：帮助用户在示例商城中快速切换商品集合，缩短找商品路径。
- 运营配置：支持运营根据商品结构和用户找品路径调整展示类目。

因此需要在运营工作台「商城业务 - 商品中心」下新增类目管理能力，初期支持展示类目和后台类目两个模块。

## 二、需求目标

1. 支持运营维护示例商城的展示类目结构，展示类目只支持 2 级，并通过映射后台类目承接商品范围。
2. 支持展示类目按城市维度配置生效范围，满足不同城市商品供给和运营策略差异。
3. 支持二级展示类目配置展示图片，用于 C 端二级入口展示；一级不配置图片。
4. 支持运营维护后台类目结构，用于商品归类和后续商品管理、属性、品牌等能力扩展。
5. 支持类目的新增、编辑、启用、禁用、删除、排序和查询，减少研发介入。
6. 保证类目数据稳定：有子类目、有关联商品或被展示类目使用的类目，不允许直接删除。
7. 支持后台类目和展示类目操作日志查询，满足运营追溯和问题排查需要。

## 三、需求详情

### 3.1 菜单入口

入口位置：运营工作台 - 商城业务 - 商品中心 - 商品管理。

在「商品管理」下新增两个独立菜单，不在同一页面内用 Tab 切换：

- 展示类目管理
- 后台类目管理

统一页面布局规则：

- 所有可编辑或可导入字段在方案中必须明确：是否必填、数据类型与允许格式、长度或数值范围、计数单位、去空格/规范化规则、默认值、唯一性、前后端校验时机与错误提示、存量数据兼容方式。不得使用“合理长度”等模糊口径，不得仅依赖组件或数据库字段限制，不得静默截断用户输入。
- 页面采用桌面端运营后台布局，筛选项按字段内容设置稳定宽度，不随大屏等比例拉长；名称、ID 等搜索框与层级、状态下拉框保持合理宽度差异。排序值、数量等短数字输入框控制在 120-180px，表格内商品排序值输入框控制在 72-96px，不随栅格等宽拉满。
- 查询区仅承载「查询」「重置」等筛选动作；操作日志、导出、批量排序和新增等页面级动作统一放在列表标题栏右侧。
- 页面工作区固定在当前桌面视口内，不使用外层页面滚动承载类目列表；左侧菜单超出高度时独立滚动。类目列表操作栏位于表格滚动区之外并保持可见，不使用会覆盖下方内容的悬浮偏移；列表卡片自适应工作区剩余高度，表格区域仅占用操作栏与分页栏之间的剩余空间并独立滚动，吸顶表头不得被操作栏遮挡；分页栏固定展示在列表卡片底部，不进入表格滚动区，也不因表格滚动到底而落到当前视口之外。操作日志列表采用相同布局，日志标题栏不得覆盖筛选区。后台类目列表在 1280px 及以上桌面视口完整展示主要字段和操作，展示类目宽表允许横向滚动。
- 弹窗设置最大视口高度，内容超出时仅弹窗主体内部滚动，标题和底部操作按钮保持可见。复杂映射、城市多选及导入预览使用整行宽度；排序值、排序间隔、类目层级和状态保持紧凑宽度；只读上下文使用标签加文本展示，不使用超长禁用输入框。
- 输入框、按钮、分页和关闭按钮需具备清晰的悬浮、键盘焦点、按下及禁用状态；状态信息不得只依赖颜色表达。

### 3.2 后台类目管理

后台类目用于商品归类和运营管理，偏稳定，不直接面向 C 端展示。

#### 3.2.1 层级规则

- 后台类目支持 3 级。
- 新建商品必须挂载到可用的后台叶子类目；当类目树只有两级时关联二级叶子类目，当存在三级时关联三级叶子类目。商品保存时落库后台类目 ID，类目名称和路径按最新类目树计算。
- 后台类目不配置图片。

#### 3.2.2 列表字段

| 字段 | 说明 |
| --- | --- |
| 类目名称 | 后台管理名称；去除首尾空格后 1-20 个字符 |
| 类目 ID | 系统生成 |
| 类目层级 | 一级 / 二级 / 三级 |
| 上级类目 | 一级为空 |
| 关联商品数 | 当前类目直接或汇总关联商品数量 |
| 排序值 | 数字越小越靠前 |
| 配置状态 | 运营配置的启用 / 禁用状态 |
| 展示状态 | 系统根据当前类目及上级类目配置状态计算的实际展示结果，轻量展示“展示中”或“不展示 + 原因” |
| 更新时间 | 最近一次保存时间 |
| 操作 | 一级、二级支持新增子类；一级、二级、三级均支持编辑、启用/禁用、删除 |

列表交互：

展示与分页：

- 默认按树形结构展示，支持展开和收起。
- 同一父级、同一层级内，默认按排序值从小到大展示；排序值越小越靠前。
- 更新时间仅作为列表字段展示，不参与类目列表默认排序。
- 列表支持分页，默认 50 条/页，支持切换每页 50 / 100 / 200 条；支持点击页码、上一页、下一页切换。
- 分页区展示总条数和当前展示范围；查询、重置、切换后台/展示类目模块后均回到第 1 页。
- 查询命中二级或三级类目时，列表额外展示其父级上下文行以保留完整路径；分页总条数、当前展示范围和每页条数仅统计实际命中行，父级上下文行不重复计数。
- 分页边界落在子类目时，下一页重复展示该子类目的父级上下文，避免页面从孤立子类目开始；重复展示的父级上下文不计入当前页条数。

树形展开：

- 点击一级类目前的箭头，展开或收起其下二级、三级类目。
- 点击二级类目前的箭头，展开或收起其下三级类目。
- 类目下没有子类目时，不展示展开/收起图标。

状态展示：

- 配置状态表示当前类目自身设置，不因上级类目禁用而自动变更。
- 展示状态为只读计算结果，不提供单独编辑入口。
- 当类目自身启用且所有上级均启用时，展示状态为“展示中”。
- 当类目自身禁用时，展示状态为“不展示”，原因展示“当前已禁用”。
- 当类目自身启用但任一上级类目禁用时，展示状态为“不展示”，原因展示“上级已禁用”；该提示使用弱样式展示，避免误解为子类目自身被禁用。
- 鼠标悬浮“上级已禁用”提示时，展示说明“上级类目已禁用，当前类目暂不在 C 端展示；上级启用后，将按当前类目状态恢复展示。”

#### 3.2.3 查询条件

- 类目名称：模糊查询。
- 类目名称查询匹配后台类目全层级名称，一级、二级、三级类目名称命中均返回。
- 查询结果保留命中类目的完整父级路径，便于运营识别所属层级。
- 输入前后空格自动去除；未输入类目名称时不按名称过滤。
- 类目 ID：精确查询；输入前后空格自动去除，未输入时不按 ID 过滤。
- 类目层级：全部、一级、二级、三级。
- 配置状态：全部、启用、禁用。

#### 3.2.4 新增和编辑

新增范围：

- 新增一级类目：填写类目名称、排序值、状态。
- 新增二级类目：系统根据当前一级类目带入所属一级类目，填写类目名称、排序值、状态。
- 新增三级类目：系统根据当前二级类目带入所属二级类目，填写类目名称、排序值、状态。
- 后台类目不展示图片上传字段。

弹窗入口：

- 点击「新增一级类目」打开新增弹窗，默认层级为一级。
- 点击「新增子类」打开新增弹窗，系统根据当前行层级自动带入下一层级，并带入当前行作为上级类目。
- 新增子类时，上级类目和类目层级置灰不可修改；如需换父级，需要回到目标父级行重新点击「新增子类」。
- 后台父类目已直接关联商品时，点击「新增子类」后在弹窗内展示非阻断提示：“当前后台类目已关联 N 个商品。新增后台子类后，已有商品关联保持不变，新商品需选择其下后台叶子类目。”该提示仅说明影响，不增加二次确认。

批量新增：

- 新增弹窗支持单个新增和批量新增。
- 批量新增只支持同一父级、同一层级下创建多个类目；不支持一次性创建跨层级类目树。
- 批量新增时使用多行文本输入类目名称，一行一个类目；系统自动去除每行首尾空格并忽略空行，每个有效类目名称需为 1-20 个字符。
- 批量新增支持设置统一状态、起始排序值和排序间隔，并按输入顺序生成排序值。
- 新增弹窗打开时，排序值默认取当前父级下同层级类目的最大排序值 + 1；若当前父级下没有同级类目，默认排序值为 1。
- 批量新增时，起始排序值默认同样取当前父级下同层级类目的最大排序值 + 1，排序间隔默认 1。
- 后台类目批量新增弹窗底部不展示额外说明文字，避免与表单字段和校验提示重复。

校验规则：

- 类目名称去除首尾空格后长度为 1-20 个字符；中文、英文、数字和常用可见符号均按 1 个字符计数，不允许换行符或控制字符进入单个类目名称。
- 单个新增和编辑时，输入框展示实时字符计数；超过 20 个字符时计数标红并拦截保存，不自动截断输入。批量新增按有效行逐条校验，存在超长名称时整批不保存；错误提示逐条展示超限行号、完整类目名称和“超过 20 个字符”的原因，多条错误分行展示。
- 原型前端、服务端接口和批量新增使用同一校验口径；系统不得静默截断。历史存量名称即使超过 20 个字符仍可正常展示，但再次编辑保存时必须调整至规则范围内。
- 类目名称全局不可重复；单个新增、批量新增、编辑保存时均需校验整棵类目树内是否已存在同名类目。
- 单个新增或编辑时，若填写的类目名称与任一已有类目重复，拦截保存并提示“类目名称全局重复，请调整后保存”。
- 批量新增时，需同时校验批量输入内部不可重复、批量输入名称不可与已有任一类目重复；如重复，拦截保存并提示重复类目名称。
- 同一父级、同一层级下排序值必须唯一；单个新增、批量新增、编辑保存时均需校验，不允许产生重复排序值。
- 单个新增时，若填写的排序值与同一父级、同一层级下已有类目重复，拦截保存并提示“同一父级、同一层级下排序值不能重复，请调整后保存”。
- 批量新增时，系统按起始排序值和排序间隔生成每条类目的排序值；生成结果需全部在 0-9999 范围内，且不能与同一父级、同一层级下已有类目排序值重复，否则拦截保存。

编辑规则：

- 点击「编辑」打开编辑弹窗，回填当前类目名称、上级类目、层级、排序值和状态。
- 编辑时，上级类目和类目层级置灰不可修改；调整上级类目或层级需通过独立「移动」入口处理。
- 编辑保存时，系统校验当前排序值是否与同一父级、同一层级下其他类目重复；如重复，拦截保存并提示调整。
- 保存时校验类目名称去除首尾空格后为 1-20 个字符、类目名称全局唯一、排序值为 0-9999 的整数、同一父级同一层级排序值不重复；校验不通过时在字段下方展示错误提示。
- 排序值标题右侧展示问号说明入口；鼠标悬浮时完整展示提示“数字越小排序越靠前；同一父级、同一层级内生效；按 1 递增，符合业务排序习惯”。

保存后反馈：

- 新增保存成功后，系统在当前列表中新增对应类目数据，并按同一父级、同一层级排序值升序重新展示，刷新分页统计，自动跳转到新增类目所在页，并短暂高亮该行。
- 批量新增保存成功后，系统按输入顺序新增多条类目数据，按起始排序值和排序间隔生成排序值，并按同一父级、同一层级排序值升序重新展示，刷新分页统计，自动跳转到本次新增类目中排序最靠前的一条所在页，并短暂高亮当前页内本次新增的类目行。
- 编辑保存成功后，系统更新当前列表行的类目名称、排序值、配置状态、展示状态、更新时间等字段；若排序值变化，当前父级下同层级列表需按排序值升序重新展示；系统自动跳转到该类目调整后所在页，短暂高亮该行，关闭弹窗并提示“类目修改成功，已按排序值更新展示位置”。
- 后台子类目创建和后台父类目叶子状态变更需在同一事务内提交。若后台子类目先创建成功，之后提交的新商品不得再关联原后台父类目；若商品先保存成功，则该商品作为原后台父类目的存量直接关联商品保留。

#### 3.2.5 移动类目

入口与定位：

- 类目行展示「移动」入口，用于调整类目的上级类目和层级。
- 普通编辑弹窗不支持直接修改类目层级和上级类目，避免名称、排序、状态编辑与树结构调整混在同一流程。
- 移动不采用拖拽改变层级；类目较多时通过「目标上级类目」搜索选择完成移动。
- 目标上级选择器展示可选父级的完整路径，支持按类目名称和完整路径搜索，并排除当前类目自身及其子孙类目。
- 打开目标上级选择器时默认展示全部合法候选，不以当前上级名称作为默认搜索条件；移动二级类目时，应展示所有可承接该子树的一级类目，不限于当前上级。
- 目标上级候选需结合当前类目子树深度过滤；若移动到某个目标上级后会导致当前类目或任一子类目超过后台类目最大三级，则该目标上级不展示为可选项。
- 目标上级选中值需按类目 ID 或等价稳定标识保存，不按类目名称匹配，避免不同分支出现同名类目时移动到错误父级。
- 在目标上级输入框中搜索时，仅过滤候选项，不自动改写已选目标；只有点击候选项后才更新移动后层级、排序默认值和影响预览。搜索无结果时展示“无可选目标上级”，失焦后输入框回显当前已选目标完整路径。

移动规则：

- 类目层级不支持手填，由目标上级类目自动推导。
- 目标上级为「-」时，当前类目移动为一级类目。
- 目标上级为一级类目时，当前类目移动为二级类目。
- 目标上级为二级类目时，当前类目移动为三级类目。
- 不允许选择当前类目自身或其任一子孙类目作为目标上级，避免形成循环层级。
- 当前类目的全部子类目随当前类目一起移动，子类目层级按移动前后的层级差整体平移。

层级校验：

- 后台类目最大支持三级。
- 移动前需计算当前类目的子树深度，公式为：移动后最深层级 = 目标层级 + 当前子树深度。
- 若移动后最深层级超过三级，拦截保存，并提示“移动后子树层级超过系统最大支持层级”。
- 示例：一级 A 下有二级 B、三级 C，将 A 移动到另一个一级类目下会导致 C 变为四级，需拦截。

名称和排序校验：

- 移动不修改类目名称，但保存时仍需校验类目名称全局不可重复。
- 移动后排序值在目标父级和目标层级范围内生效，默认取目标父级下同层级最大排序值 + 1。
- 运营可在移动弹窗中调整移动后排序值；保存时校验排序值为 0-9999 的整数，且目标父级同层级内排序值唯一。

关联影响：

- 移动后类目完整路径、面包屑、查询结果路径、导出路径和操作日志对象路径需按最新树结构展示。
- 后台类目跨级移动不生成新的后台类目 ID；已关联商品保存的后台类目 ID 不变，不自动迁移到其他后台类目，也不自动下架，关联商品数不因移动清零。
- 后台类目移动后，商品侧后台类目路径按最新树结构刷新；已被展示类目（前台类目）映射引用的后台类目仍按后台类目 ID 保留映射，映射路径文案同步刷新。
- 后台类目移动本身不改变展示类目（前台类目）的映射 ID；系统按移动后的最新后台类目父子关系重新计算展示类目商品池，不按旧路径字符串判断商品归属。
- 被移动的后台叶子类目后续新增后台子类目时，原后台类目变为非叶子类目：已有商品继续保留原后台类目关联，不自动迁移或下架；新建商品、复制商品或主动修改商品类目时不得再选择该后台非叶子类目。
- 原父级若移动后不再有子类目，列表需刷新展开箭头和删除限制判断。
- 移动后的类目进入新父级的批量排序范围，并从原父级的批量排序范围移除。

保存后反馈：

- 移动保存成功后，系统同步更新当前类目及其子类目的层级、上级类目、排序值、更新时间、完整路径和展示状态。
- 保存成功后列表按最新树结构和排序值重新展示，跳转到移动后类目所在页并高亮移动子树。
- 移动操作写入操作日志，记录原上级、目标上级、原层级、目标层级、受影响子类目数量和排序值变化。
- 移动保存失败时，本次移动整体失败，后台列表保持保存前结构。

#### 3.2.6 删除规则

删除限制：

- 存在子类目时，不允许删除。
- 后台类目直接关联商品数大于 0 时不允许删除；其中包括后台叶子类目新增子类目后仍保留在原后台类目上的存量商品。
- 已被展示类目或其他配置引用时，不允许删除。
- 点击「删除」时需要二次确认。

拦截提示：

- 若类目存在子类目，系统提示“该类目下存在子类目，不允许删除。请先处理子类目。”，不执行删除。
- 若后台类目关联商品数大于 0，系统提示该类目关联商品，不允许删除。
- 若后台类目已被展示类目映射引用，系统提示“该后台类目已被展示类目「XX」映射引用，不允许删除。”，不执行删除。

删除成功：

- 若类目允许删除，确认后从列表移除，刷新分页统计和父级展开状态，并提示删除成功。

#### 3.2.7 批量排序

排序范围：

- 支持按同一父级类目下的子类目批量调整排序。
- 一级类目之间排序、同一一级下的二级类目排序、同一二级下的三级类目排序分别独立处理。
- 点击「批量排序」打开排序弹窗，系统根据当前后台类目列表实时生成可排序范围，选择排序范围后展示该父级下真实可排序类目列表。

弹窗交互：

- 排序弹窗打开时，列表带出当前类目的原始排序值。
- 排序弹窗支持拖拽调整顺序；拖动完成后弹窗内序号和排序值自动按新顺序刷新，排序值按 1 为间隔重新生成。
- 排序弹窗底部规则分两行展示：第一行说明“排序只支持同一父级下调整；弹窗打开时带出当前列表中的原始排序值。”，第二行说明“拖拽调整后，系统按 1 为间隔重新生成排序值；保存后刷新列表顺序。”
- 不支持跨层级、跨父级排序。

保存后反馈：

- 保存后更新排序值、更新时间和列表行顺序，列表按拖拽后的新顺序重新展示。

#### 3.2.8 启用/禁用规则

定义与业务影响：

- 禁用后台类目表示停止该后台类目的新增商品绑定和类目渠道展示，不等同于下架商品；如需商品全渠道不可售，应使用商品下架能力。
- 禁用后台类目后，该后台类目不可用于新建商品、复制商品或主动修改商品类目；已关联商品保留原后台类目 ID、后台类目路径和商品自身上下架状态，不自动迁移、不自动下架。
- 禁用后台父类目时，其全部后台子孙类目实际不可用，但不修改后台子孙类目自身的配置状态。父级恢复启用后，后台子孙类目按自身启用/禁用状态恢复。
- 被禁用后台类目直接关联的商品及其后台子孙类目下商品暂时退出相关展示类目（前台类目）商品池；展示类目映射关系和商品排序配置保留，不视为取消映射。
- 禁用后台类目只影响类目选择和展示类目商品池；商品是否能通过其他非类目渠道访问或交易，继续由商品自身状态、发布渠道和对应渠道规则决定。

状态展示：

- 后台类目列表通过“配置状态”和“展示状态”区分自身配置与实际生效结果：当前后台类目自身禁用时展示“禁用”“不展示 / 当前已禁用”；后台子类目自身启用但受父级禁用影响时展示“启用”“不展示 / 上级已禁用”。
- 后台类目的关联商品数、后台子类目结构和展示类目映射数量不因禁用清零。

操作反馈：

- 点击启用或禁用时需要二次确认。
- 禁用确认弹窗需展示受影响后台子孙类目数、关联商品数和展示类目（前台类目）数，并明确说明：已有商品不自动下架，展示类目映射和商品排序保留。
- 禁用成功后，在同一事务内更新当前后台类目配置状态及后台类目子树实际生效状态，并重新计算受影响展示类目的商品池、展示状态和兜底图片。
- 禁用后台类目后仍允许运营改名、调整排序或移动该后台类目；后台类目 ID 和商品关联 ID 不变，恢复启用后按最新后台类目树生效。
- 禁用后台类目后，仍按既有规则限制删除：存在后台子类目、关联商品或展示类目映射时不得删除。

恢复启用：

- 后台类目恢复启用后，后台子孙类目按自身配置状态恢复；符合条件的商品按原后台类目关联重新进入相关展示类目商品池，并恢复原商品排序值。
- 展示类目的展示状态和未配置类目图片时的 C 端兜底商品图需重新计算；运营配置的展示类目图片不变。
- 恢复启用不自动修改商品自身上下架状态：原上架商品仍为上架，原下架商品仍为下架。

#### 3.2.9 商家创建商品选择类目

适用页面：

- 商家后台创建商品页，字段为「商品类目」。

展示规则：

- 商品类目选择框读取运营配置的后台类目。
- 类目选择数据以后台类目 ID 为稳定主键，读取当前后台类目的父子关系、层级、排序值、配置状态和实际生效状态；不得在商家创建商品页维护一套独立类目字典。
- 商品类目选择框默认收起，未选择时框内提示“请选择商品叶子类目”。
- 商品类目字段下方不展示额外说明文案。
- 后台类目最多支持 3 级，商家创建商品时必须选择当前类目树中可用的叶子类目，否则不允许确认或保存；叶子类目指当前没有任何子类目的后台类目。
- 当某条类目路径只有两级时，二级类目即为叶子类目，允许商品直接关联该二级类目；当二级类目下存在三级类目时，二级类目为非叶子类目，只用于展开和导航，商品必须继续选择三级叶子类目。
- 类目列表按后台类目的同级排序值展示，排序值越小越靠前。
- 商家创建商品选择类目时，不展示后台类目的关联商品数。
- 父级类目禁用、当前类目禁用或展示状态为“不展示”的类目，在商家创建商品页不可选择。
- 不可选择类目可弱展示禁用原因，避免商家误以为系统无类目。
- 商家创建商品的级联列表和搜索结果仍展示已禁用后台类目的最新完整路径，但置灰不可选择；当前后台类目自身禁用时提示“当前后台类目已禁用”，受父级影响时提示“上级后台类目已禁用”。
- 二级类目没有任何三级子类目时，该二级类目作为叶子类目可直接确认选择，三级列保持空白，不展示额外说明文案。
- 二级类目存在三级子类目但无可用三级类目时，二级类目不能作为叶子类目选择，三级列展示“暂无可用叶子类目”，不能确认选择。
- 最近使用类目展示在搜索框下方，仅展示最近，不展示常用或热门。
- 最近使用类目使用统一前缀展示，格式为“最近：类目A、类目B、类目C”；不在每个类目前重复展示“最近”。
- 最近使用类目按当前商家账号维度记录，取该账号最近成功保存商品时选择过的叶子类目，叶子类目可能为一级、二级或三级。
- 最近使用类目最多展示 3 个，按最近使用时间倒序排列。
- 若最近使用的后台类目已被禁用、上级后台类目被禁用、已删除或已变为非叶子类目，则不在最近列表中展示。
- 最近使用类目按后台类目 ID 保存，展示名称和完整路径每次按最新类目树计算；后台类目改名或移动后，最近使用入口不得保留旧路径。

交互规则：

- 选择框最多支持三级级联选择，按一级类目、二级类目、三级类目分列展示；当前层级已经是叶子类目时可直接选择，不要求继续补齐后续层级。
- 点击「商品类目」选择框时展开类目选择面板；面板已展开时再次点击选择框则收起面板。
- 类目选择面板展开后必须完整落在当前桌面视口的可视区域内；空间不足时页面自动滚动定位，保证各级类目列表、已选路径和确认按钮可操作。
- 点击一级或二级类目后，底部「已选」区域需跟随当前级联选择实时变化；若当前类目为可用叶子类目，直接选中当前类目；若当前类目为非叶子类目，默认预选其子树中排序最靠前的可用叶子类目。
- 支持按类目名称搜索，一级、二级、三级类目命中时均返回完整类目路径。
- 搜索结果展示最新完整路径，例如“手机数码与电器 / 手机_小米”或“汽车清洁与保养 / 洗车消耗品 / 洗车消耗品商品”，商家点击可直接选中可用的叶子类目；非叶子类目只展示路径，不允许作为商品类目确认。
- 支持展示最近使用类目，帮助商家快速选择历史高频类目。
- 已选中后，输入框回显完整类目路径。
- 若商家未选择可用叶子类目，保存商品时拦截并提示“请选择商品叶子类目”。
- 点击「确认选择」和点击商品「保存」时均需基于后台类目最新状态重新校验：所选类目仍存在、仍为叶子类目、当前类目及全部上级均启用；任一条件不满足时清空无效选择并提示重新选择。
- 商家搜索后台非叶子类目时仍可展示其最新完整路径，但只用于定位和展开，不允许确认选择；最近使用和搜索结果均按同一后台叶子类目有效性规则过滤。

保存与跨模块联动：

- 商品保存时落库所选后台叶子类目 ID，不以类目名称或路径字符串作为关联键；商品列表和详情中的类目路径按当前后台类目树实时展示。
- 已关联商品的后台叶子类目后续新增后台子类目时，原后台类目变为非叶子类目：已有商品关联和上下架状态不自动变更；展示类目（前台类目）继续按映射后台类目 ID 及最新父子关系计算商品池。该原后台类目不再允许新商品选择。
- 商家编辑存量商品时，若商品保存的后台类目 ID 未发生变化，允许保存商品名称、价格、库存等非类目字段，不强制迁移历史关联；商家主动修改商品类目、复制创建商品或重新选择商品类目时，必须选择当前可用的后台叶子类目。
- 商家确认或保存商品时若所选后台类目刚被新增子类目、已变为非叶子类目，拦截提交并提示“所选后台类目已变为非叶子类目，请重新选择商品类目。”并刷新后台类目树；若商品保存事务先于后台子类目创建事务完成，则该商品按存量直接关联规则保留。
- 后台类目改名或移动时，商品关联的后台类目 ID 不变；商品类目路径、商家创建商品选择器和展示类目商品池按最新树结构刷新。
- 后台类目禁用时，不自动修改已有商品的上下架状态和类目关联，但该类目及受其影响的子类目不可用于新建商品；对应商品在类目恢复可用前不进入 C 端展示类目商品池。
- 商家商品列表和商品详情继续展示存量商品原后台类目完整路径，并在类目旁标记“后台类目已禁用”或“上级后台类目已禁用”；商品自身状态继续独立展示“上架”或“下架”，不得将后台类目禁用展示成商品已下架。
- 商家编辑关联无效后台类目的存量商品时，只修改商品名称、价格、库存和详情等非类目字段允许保存；主动修改商品类目时必须选择当前有效的后台叶子类目。
- 已下架商品重新上架时需校验其关联后台类目仍为有效后台叶子类目；后台类目已禁用、受上级禁用影响或已变为非叶子类目时，拦截上架并提示“原后台类目已不可用，请先更换为有效后台叶子类目”。
- 商家已选择后台类目但尚未保存时，如运营先禁用该后台类目，商品保存需基于最新后台类目状态拦截，并提示“所选后台类目已不可用，请重新选择商品类目”。若商品保存事务先完成，则该商品作为存量关联保留。
- 已产生订单不受后台类目禁用影响，履约、退款和售后继续按订单与商品既有规则处理。
- 商品成功保存、上下架状态变化或后台类目状态变化后，系统需按后台类目 ID 重新计算受影响展示类目的有效商品池和展示状态。

#### 3.2.10 导出规则

导出范围：

- 后台类目管理支持按当前筛选条件导出。
- 单次导出上限 1000 条；超过 1000 条时，系统提示缩小筛选范围后再导出。

导出内容：

- 导出格式按层级拆列：一级类目、二级类目、三级类目。
- 导出字段顺序：一级类目、二级类目、三级类目、类目 ID、类目层级、上级类目、关联商品数、排序值、配置状态、展示状态、更新时间。

导出交互：

- 点击「导出」后需要二次确认，确认后生成导出文件。

#### 3.2.11 操作日志

入口与页面：

- 后台类目管理支持查看操作日志，入口位于类目列表标题栏右侧，与导出、批量排序和新增等页面级动作同区展示。
- 点击「操作日志」后进入独立的「后台类目操作日志」页面，不使用弹窗承接。
- 操作日志页支持点击「返回后台类目管理」回到后台类目列表页。
- 日志页顶部右侧展示轻量的「返回后台类目管理」按钮；筛选条件单独成行，「查询」「重置」位于筛选项下方一行并左对齐。

日志范围与字段：

- 日志记录范围：新增、编辑、移动、删除、启用、禁用、批量排序、导出。
- 日志字段：操作时间、操作类型、操作人、操作对象、变更内容、操作结果。
- 操作对象展示类目名称和类目 ID；批量排序按排序范围记录，如“一级类目”“车载数码下的二级类目”。
- 变更内容需说明本次操作的关键变化，例如新增位置、状态变化、排序范围、导出范围。
- 后台父类目已有商品时新增后台子类目，日志需记录后台父类目 ID 和路径、新增后台子类目信息、后台父类目由叶子变为非叶子、受影响存量商品数、操作人和操作时间。
- 启用或禁用后台类目时，日志需记录受影响后台子孙类目数、关联商品数、展示类目（前台类目）数，并标记商品未自动下架、映射和商品排序是否保留。
- 日志默认按操作时间倒序展示。

筛选与分页：

- 支持按操作对象模糊查询，匹配类目名称或类目 ID。
- 支持按操作类型筛选：全部、新增、编辑、移动、删除、启用、禁用、排序、导出。
- 支持按操作人筛选：全部或具体操作人。
- 日志列表支持分页，默认 50 条/页，支持切换每页 50 / 100 / 200 条。
- 查询结果为空时展示空状态，不影响原后台类目列表。

### 3.3 展示类目管理

展示类目用于 C 端展示和运营编排，可按城市维度控制生效范围。

#### 3.3.1 层级规则

- 展示类目只支持 2 级。
- C 端展示一级类目和二级类目。
- 一级类目展示为顶部 Tab，如数码、穿搭、养护、食品。
- 二级类目展示为一级 Tab 下的图片入口，如 OPPO、华为、运动鞋、短袖。
- 展示类目不支持三级，不提供新增三级类目入口。
- 推荐、热销、新品、活动等营销策略不属于类目体系，前后台类目管理均不提供相关配置；如需配置，由独立营销策略能力承接。

#### 3.3.2 后台类目映射规则

映射基础规则：

- 展示类目不直接承接商品，需通过映射后台类目确定商品范围。
- 展示类目商品池按后台类目 ID 及当前父子关系计算：商品直接挂载于已映射后台类目，或挂载于该后台类目的任一子孙类目时，均属于该展示类目商品池；不得依赖后台类目路径字符串前缀匹配。
- 已映射后台类目由叶子类目变为非叶子类目后，展示类目（前台类目）商品池同时包含仍直接关联该后台父类目的存量商品，以及其全部有效后台子孙类目下的商品。
- 新增或编辑展示类目时必须选择至少一个映射后台类目。
- 映射后台类目的选项来自后台类目管理中已配置的后台类目。
- 展示一级、二级类目均支持配置映射后台类目；二级类目可在所属一级类目的基础上进一步细分展示范围。

映射关系：

- 支持 1:1 映射：一个展示类目映射一个后台类目。
- 支持 1:N 映射：一个展示类目映射多个后台类目，用于将多个后台商品范围合并展示到同一展示入口。
- 支持 N:1 映射：多个展示类目映射同一个后台类目，用于同一后台商品范围在不同展示运营入口复用。
- 后台类目改名不改变映射和商品归属；后台类目移动后，系统按最新父子关系重新判断商品是否仍落在各展示类目的映射范围内，并同步刷新映射路径文案。
- 后台类目禁用后不自动解除已有展示类目（前台类目）映射；映射选择器保留历史已选项并标记“已禁用”，但禁用后台类目不可作为新增映射重新选择。
- 展示类目只映射一个后台类目且该后台类目实际不可用时，展示状态为“不展示 / 映射类目不可用”；映射多个后台类目时，只排除不可用后台类目范围内的商品，其他有效后台类目仍有有效商品时展示类目继续展示。

选择器能力：

- 映射后台类目选择器支持多选、名称搜索、按后台类目层级筛选和已选项汇总展示，避免后台类目数量较大时只能滚动查找。

#### 3.3.3 城市维度配置

配置目标：

- 展示类目支持配置展示地区，用于控制该类目在哪些城市的 C 端页面展示和生效。
- 城市维度只影响展示类目的 C 端展示、商品查询和运营编排，不改变后台类目结构，也不影响商家创建商品时选择后台类目。
- 城市配置以展示类目自身为准，并与上级类目、配置状态、映射后台类目和商品有效状态共同计算最终展示结果。

展示地区：

- 展示地区通过同一个「展示地区」选择框配置，选项包含「全国」和具体城市。
- 默认选中「全国」，表示不按城市限制展示。
- 「全国」与具体城市互斥：选择「全国」时不支持同时选择其他城市；选择任一具体城市时，系统自动取消「全国」。
- 具体城市支持多选；未选择「全国」且未选择任何具体城市时，不允许保存。
- 城市选项来自平台已开通且可运营的城市基础数据，只展示启用城市；城市名称和城市 ID 以城市基础数据为准。
- 城市选择器支持按城市名称、城市 ID、拼音或首字母搜索，支持按省份筛选，支持已选城市汇总和清空。
- 城市选择框搜索时仅过滤候选城市，不直接改写已选值；搜索无结果时展示“无匹配城市”。支持点击候选项或回车选择当前第一个可见候选项。
- 列表筛选城市失焦后需回显当前已选城市或「全国」，未确认选择的临时输入不作为筛选条件生效；若输入内容能精确匹配城市名称或「全国」，按该城市或「全国」归一为有效筛选条件。

父子联动：

- 一级展示类目配置展示地区后，其下二级展示类目的最终展示地区必须是「一级类目展示地区」与「二级类目展示地区」的交集。
- 当一级类目为「全国」时，二级类目按自身展示地区生效。
- 当二级类目为「全国」时，二级类目继承一级类目的展示地区生效。
- 当一级和二级均配置具体城市时，二级类目仅在两者交集城市生效。
- 若交集为空，二级类目展示状态为“不展示 / 展示地区无交集”，但不自动修改配置状态和展示地区。
- 编辑一级类目展示地区时，系统需实时提示受影响的二级类目数量；保存后重新计算所有子类目的展示状态和地区覆盖范围。

展示状态计算：

- 展示状态由配置状态、上级状态、展示地区、映射后台类目状态和有效商品共同决定。
- 当类目自身禁用时，展示状态为“不展示 / 当前已禁用”。
- 当任一上级类目禁用时，展示状态为“不展示 / 上级已禁用”。
- 当展示地区为空或父子城市交集为空时，展示状态为“不展示 / 展示地区无交集”。
- 当映射后台类目全部不可用、被删除或禁用时，展示状态为“不展示 / 映射类目不可用”。
- 当映射后台类目下无有效商品时，C 端不展示该类目入口，后台列表展示“不展示 / 无有效商品”。
- 同时存在多个不展示原因时，原因优先级为：当前已禁用 > 上级已禁用 > 展示地区无交集 > 映射类目不可用 > 无有效商品。

数据准确性：

- 展示类目保存时需落库类目基础信息、映射后台类目、展示地区、排序值、状态和操作人，所有字段在同一事务内提交。
- 展示地区保存成功后立即进入 C 端读取结果；保存过程中需保证原子性，避免 C 端读取到不完整数据。
- C 端请求展示类目时必须带入当前城市；服务端按城市过滤后返回一级和二级类目树。
- 服务端返回时需剔除无有效二级入口的一级类目，避免 C 端展示空 Tab。
- 城市基础数据发生城市停用、合并或更名时，展示类目保留城市 ID，列表按最新城市名称展示；已停用城市在配置中弱提示“不再可选”，但历史日志保留原城市 ID 和当时名称。
- 若类目展示地区、后台类目映射或商品有效状态任一条件变化，需重新计算对应展示类目的展示结果，保证 C 端与后台展示状态一致。

#### 3.3.4 图片规则

配置范围：

- 一级类目不需要配置图片。
- 二级类目支持配置类目图片，但不强制上传。
- 新增或编辑二级类目时展示图片上传字段；新增或编辑一级类目时不展示图片字段。
- 二级类目未上传图片时允许保存和启用。
- 二级类目图片已上传后，支持更新图片和删除图片。
- 二级类目有效头图优先级为：运营配置的类目图片 > 当前展示类目有效商品池中排序第一商品的主图。
- 运营未配置类目图片时，系统自动读取该展示类目下排序第一的有效商品主图作为二级类目头图；“第一商品”按商品排序值从小到大排序，排序值相同时按后端默认顺序兜底。
- 删除运营配置图片并保存后，C 端立即恢复读取排序第一的有效商品主图；配置后台的「类目图片」字段为空值，列表以「-」展示。
- 若未配置类目图片，且类目下无有效商品或排序第一商品无可用主图，则 C 端不展示类目头图；类目仍按“无有效商品”等既有展示状态规则处理，不新增图片必填校验。

上传与更新：

- 未上传图片时，图片区域展示「+」上传入口；点击「+」必须打开本地图片选择。
- 已上传图片时，图片区域展示当前图片预览；点击图片预览打开图片放大查看层，不打开文件选择。
- 已上传图片后展示「更新图片」「删除图片」操作。
- 点击「更新图片」打开本地图片选择；用户选择新图片后，图片区域保持「已上传」状态，并提示图片更新成功。
- 点击「删除图片」先展示二次确认；用户确认后清空当前图片，图片区域回到「+」上传入口，并提示图片已删除。
- 点击「+」上传或「更新图片」时，只有用户完成图片选择后才展示上传或更新成功；用户取消文件选择时不改变当前图片状态。

列表展示与预览：

- 配置后台列表「类目图片」字段只展示运营已配置的类目图片，不展示系统兜底的商品图；未配置时展示「-」，避免业务误认为商品图已被人工配置。
- 运营配置图片时，列表展示最新上传或更新后的类目图片缩略图；点击缩略图打开图片放大查看层。未配置图片时不提供放大入口。
- C 端独立按有效头图优先级取图。商品排序、商品有效状态、后台类目映射、后台类目新增子类或商品迁移后台类目导致展示类目商品池变化后，系统需重新计算 C 端兜底图片，但不写回类目图片配置、不改变配置后台列表和编辑弹窗的未配置状态。
- 图片放大查看层需展示原图等比预览，支持点击关闭按钮、点击遮罩或按 Esc 关闭。

#### 3.3.5 列表字段

| 字段 | 说明 |
| --- | --- |
| 类目名称 | C 端展示名称；去除首尾空格后 1-20 个字符 |
| 类目 ID | 系统生成 |
| 类目层级 | 一级 / 二级 |
| 上级类目 | 一级为空，二级展示所属一级 |
| 映射后台类目 | 当前展示类目映射的一个或多个后台类目 |
| 展示地区 | 全国或具体城市；选择具体城市时展示城市数量和城市摘要 |
| 类目图片 | 配置后台仅展示二级类目运营已配置的图片缩略图；未配置时展示「-」，不展示 C 端运行时兜底的商品图。一级展示「-」 |
| 排序值 | 数字越小越靠前 |
| 配置状态 | 运营配置的启用 / 禁用状态 |
| 展示状态 | 系统根据当前类目及上级类目配置状态计算的实际 C 端展示结果，轻量展示“展示中”或“不展示 + 原因” |
| 更新时间 | 最近一次保存时间 |
| 操作 | 一级支持新增子类；一级、二级均支持编辑、启用/禁用、删除 |

列表交互：

展示与分页：

- 默认按树形结构展示，支持展开和收起。
- 同一父级、同一层级内，默认按排序值从小到大展示；排序值越小越靠前。
- 更新时间仅作为列表字段展示，不参与类目列表默认排序。
- 列表支持分页，默认 50 条/页，支持切换每页 50 / 100 / 200 条；支持点击页码、上一页、下一页切换。
- 分页区展示总条数和当前展示范围；查询、重置、切换后台/展示类目模块后均回到第 1 页。
- 查询命中二级展示类目时，列表额外展示其一级父级上下文行以保留完整路径；分页总条数、当前展示范围和每页条数仅统计实际命中行，父级上下文行不重复计数。
- 分页边界落在二级展示类目时，下一页重复展示其一级父级上下文，避免页面从孤立二级类目开始；重复展示的一级父级上下文不计入当前页条数。

树形展开：

- 点击一级类目前的箭头，展开或收起其下二级类目。
- 类目下没有子类目时，不展示展开/收起图标。

状态展示：

- 配置状态表示当前类目自身设置，不因上级类目禁用而自动变更。
- 展示状态为只读计算结果，不提供单独编辑入口。
- 当类目自身启用且所有上级均启用时，展示状态为“展示中”。
- 当类目自身禁用时，展示状态为“不展示”，原因展示“当前已禁用”。
- 当类目自身启用但任一上级类目禁用时，展示状态为“不展示”，原因展示“上级已禁用”；该提示使用弱样式展示，避免误解为子类目自身被禁用。
- 鼠标悬浮“上级已禁用”提示时，展示说明“上级类目已禁用，当前类目暂不在 C 端展示；上级启用后，将按当前类目状态恢复展示。”

#### 3.3.6 查询条件

- 类目名称：模糊查询。
- 类目名称查询匹配展示类目全层级名称，一级、二级类目名称命中均返回。
- 查询结果保留命中类目的完整父级路径，便于运营识别所属层级。
- 输入前后空格自动去除；未输入类目名称时不按名称过滤。
- 类目 ID：精确查询；输入前后空格自动去除，未输入时不按 ID 过滤。
- 类目层级：全部、一级、二级。
- 配置状态：全部、启用、禁用。
- 所属父级：选项来自当前展示类目树中真实存在的一级类目；选择具体父级后，仅返回该父级下的二级类目，一级类目因无直接父级不展示。
- 展示地区：全部、全国、具体城市。选择「具体城市」时，查询配置了具体城市的类目。
- 查询区「查询」「重置」位于筛选项下方一行并左对齐。

#### 3.3.7 新增类目

新增一级类目：

- 填写类目名称、映射后台类目、展示地区、排序值、状态。
- 不展示图片字段。

新增二级类目：

- 系统根据当前一级类目带入所属一级类目。
- 填写类目名称、映射后台类目、展示地区、上传类目图片、排序值、状态。
- 类目图片非必填，未上传图片不拦截保存和启用。

新增交互：

入口与层级：

- 点击「新增一级类目」打开新增弹窗，默认层级为一级。
- 点击一级类目行的「新增子类」打开新增二级类目弹窗，并带入当前行作为上级类目。
- 二级类目不展示「新增子类」入口。
- 新增子类时，上级类目和类目层级置灰不可修改；如需换父级，需要回到目标父级行重新点击「新增子类」。

单个/批量新增：

- 新增弹窗支持单个新增和批量新增。
- 批量新增只支持同一父级、同一层级下创建多个类目；不支持一次性创建跨层级类目树。
- 批量新增时使用多行文本输入类目名称，一行一个类目；系统自动去除每行首尾空格并忽略空行，每个有效类目名称需为 1-20 个字符。
- 批量新增支持设置统一状态、起始排序值和排序间隔，并按输入顺序生成排序值。
- 新增弹窗打开时，排序值默认取当前父级下同层级类目的最大排序值 + 1；若当前父级下没有同级类目，默认排序值为 1。
- 批量新增时，起始排序值默认同样取当前父级下同层级类目的最大排序值 + 1，排序间隔默认 1。
- 新增弹窗默认使用「单个新增」，状态默认「启用」。

映射、城市与图片：

- 新增展示类目时，映射后台类目默认不选中任何选项，需运营主动选择后才能保存，避免默认映射导致误配置。
- 新增或编辑二级展示类目时，可选择的映射后台类目必须受上级一级展示类目的映射范围约束；仅允许选择上级一级展示类目已映射后台类目本身及其子孙后台类目。
- 若上级一级展示类目未配置映射后台类目，则二级展示类目不可保存，需先补齐上级一级展示类目的映射后台类目。
- 示例：一级展示类目映射「车载数码」时，其二级展示类目只能映射「车载数码」及其下级后台类目，不允许映射「服饰商品」下的后台类目。
- 新增展示类目时，展示地区默认选择「全国」；选择具体城市时支持多选，并自动取消「全国」。
- 新增二级展示类目时，若上级一级类目配置了具体城市，系统需提示“二级类目最终展示地区受上级类目限制”。
- 类目名称去除首尾空格后长度为 1-20 个字符；中文、英文、数字和常用可见符号均按 1 个字符计数，不允许换行符或控制字符进入单个类目名称。
- 单个新增和编辑时，输入框展示实时字符计数；超过 20 个字符时计数标红并拦截保存，不自动截断输入。批量新增按有效行逐条校验，存在超长名称时整批不保存；错误提示逐条展示超限行号、完整类目名称和“超过 20 个字符”的原因，多条错误分行展示。
- 原型前端、服务端接口和批量新增使用同一校验口径；系统不得静默截断。历史存量名称即使超过 20 个字符仍可正常展示，但再次编辑保存时必须调整至规则范围内。
- 类目名称全局不可重复；单个新增、批量新增、编辑保存时均需校验整棵展示类目树内是否已存在同名类目。
- 单个新增或编辑时，若填写的类目名称与任一已有展示类目重复，拦截保存并提示“类目名称全局重复，请调整后保存”。
- 批量新增时，需同时校验批量输入内部不可重复、批量输入名称不可与已有任一展示类目重复；如重复，拦截保存并提示重复类目名称。
- 同一父级、同一层级下排序值必须唯一；单个新增、批量新增、编辑保存时均需校验，不允许产生重复排序值。
- 单个新增时，若填写的排序值与同一父级、同一层级下已有类目重复，拦截保存并提示“同一父级、同一层级下排序值不能重复，请调整后保存”。
- 批量新增时，系统按起始排序值和排序间隔生成每条类目的排序值；生成结果需全部在 0-9999 范围内，且不能与同一父级、同一层级下已有类目排序值重复，否则拦截保存。
- 展示二级类目弹窗展示图片上传入口；上传成功后展示已上传状态。
- 图片未上传时展示「+」上传入口，点击后打开本地图片选择。
- 图片已上传后展示图片预览和「更新图片」「删除图片」操作；点击图片预览打开图片放大查看层。
- 展示一级类目弹窗不展示图片上传入口。
- 展示二级类目批量新增时状态默认「启用」，切换单个/批量新增模式不自动改写当前状态；如需暂不展示，由运营主动选择「禁用」。图片允许后续逐个编辑补充，未上传图片不拦截启用。

保存校验：

- 保存时校验类目名称去除首尾空格后为 1-20 个字符、类目名称全局唯一、映射后台类目至少选择一个、展示地区合法、排序值为 0-9999 的整数、同一父级同一层级排序值不重复；二级类目图片不做必填校验。
- 未选择「全国」且未选择任何具体城市时，拦截保存并提示“请选择至少一个展示地区”。
- 保存二级展示类目时若父子城市交集为空，允许保存但需二次确认，确认文案提示“当前二级类目与上级类目展示地区无交集，保存后暂不在 C 端展示”。
- 排序值标题右侧展示问号说明入口；鼠标悬浮时完整展示提示“数字越小排序越靠前；同一父级、同一层级内生效；按 1 递增，符合业务排序习惯”。

保存后反馈：

- 新增保存成功后，系统在当前列表中新增对应类目数据，并按同一父级、同一层级排序值升序重新展示，刷新分页统计，自动跳转到新增类目所在页，并短暂高亮该行。
- 批量新增保存成功后，系统按输入顺序新增多条类目数据，按起始排序值和排序间隔生成排序值，并按同一父级、同一层级排序值升序重新展示，刷新分页统计，自动跳转到本次新增类目中排序最靠前的一条所在页，并短暂高亮当前页内本次新增的类目行。

#### 3.3.8 编辑类目

可编辑范围：

- 支持修改类目名称、映射后台类目、展示地区、排序值、状态。
- 二级类目支持修改类目图片。
- 不支持通过普通编辑直接改变类目层级或上级类目；调整上级类目或层级需通过独立「移动」入口处理。

弹窗回填：

- 点击「编辑」打开编辑弹窗，回填当前类目数据。
- 编辑弹窗中的上级类目和类目层级置灰不可修改。
- 编辑展示类目时，映射后台类目仅回填当前类目已有映射，不额外默认勾选其他后台类目。
- 编辑一级类目时不展示图片字段；编辑二级类目时展示图片字段。
- 编辑保存时，系统校验当前排序值是否与同一父级、同一层级下其他类目重复；如重复，拦截保存并提示调整。

图片编辑：

- 编辑二级类目时，图片上传区仅回填运营已配置的类目图片；商品图兜底不作为已配置图片回填，避免业务误认为已人工上传。若原类目已有配置图片，图片区域默认展示当前图片预览；点击图片预览打开图片放大查看层。
- 编辑二级类目时，点击「更新图片」打开本地图片选择，选择新图片后仅更新弹窗内图片状态，需点击保存后写入列表，并在列表「类目图片」字段展示最新图片缩略图。
- 编辑二级类目时，点击「删除图片」展示二次确认；确认后弹窗内图片状态回到「+」上传入口，需点击保存后配置后台列表图片更新为「-」；C 端恢复按排序第一有效商品的主图兜底。

保存后反馈：

- 保存成功后，系统更新当前列表行的类目名称、映射后台类目、展示地区、类目图片、排序值、配置状态、展示状态、更新时间等字段；若排序值变化，当前父级下同层级列表需按排序值升序重新展示；若展示地区变化，需同步刷新子类目的地区覆盖和展示状态；系统自动跳转到该类目调整后所在页，短暂高亮该行，关闭弹窗并提示“类目修改成功，已按排序值更新展示位置”。

#### 3.3.9 移动类目

入口与定位：

- 展示类目行展示「移动」入口，用于调整展示类目的上级类目和层级。
- 普通编辑弹窗不支持直接修改展示类目层级和上级类目，避免基础信息编辑与 C 端展示结构调整混在同一流程。
- 移动不采用拖拽改变层级；展示类目较多时通过「目标上级类目」搜索选择完成移动。
- 目标上级选择器展示可选父级的完整路径，支持按类目名称和完整路径搜索，并排除当前类目自身及其子孙类目。
- 打开目标上级选择器时默认展示全部合法候选，不以当前上级名称作为默认搜索条件；移动二级展示类目时，应展示所有合法一级展示类目，不限于当前上级。
- 目标上级候选需结合当前类目子树深度过滤；若移动到某个目标上级后会导致当前类目或任一子类目超过展示类目最大二级，则该目标上级不展示为可选项。
- 目标上级选中值需按类目 ID 或等价稳定标识保存，不按类目名称匹配，避免不同分支出现同名展示类目时移动到错误父级。
- 在目标上级输入框中搜索时，仅过滤候选项，不自动改写已选目标；只有点击候选项后才更新移动后层级、排序默认值和影响预览。搜索无结果时展示“无可选目标上级”，失焦后输入框回显当前已选目标完整路径。

移动规则：

- 展示类目层级不支持手填，由目标上级类目自动推导。
- 目标上级为「-」时，当前展示类目移动为一级展示类目。
- 目标上级为一级展示类目时，当前展示类目移动为二级展示类目。
- 展示类目最大支持二级，不支持产生三级展示类目。
- 不允许选择当前类目自身或其任一子孙类目作为目标上级，避免形成循环层级。
- 合法的层级调整仅包括：二级展示类目移动到其他一级展示类目下、二级展示类目提升为一级展示类目、无子类目的一级展示类目降为二级展示类目。
- 有二级子类目的一级展示类目不允许降为二级展示类目，否则现有子类目会形成三级展示类目；系统需直接过滤此类目标上级，并在校验时兜底拦截。

层级校验：

- 移动前需计算当前类目的子树深度，公式为：移动后最深层级 = 目标层级 + 当前子树深度。
- 若移动后最深层级超过二级，拦截保存，并提示“移动后子树层级超过系统最大支持层级”。
- 示例：一级展示类目下存在二级展示类目时，不允许将该一级类目移动到另一个一级类目下，因为子类目会变成三级展示类目。

名称、排序和城市校验：

- 移动不修改类目名称，但保存时仍需校验展示类目名称全局不可重复。
- 移动后排序值在目标父级和目标层级范围内生效，默认取目标父级下同层级最大排序值 + 1。
- 运营可在移动弹窗中调整移动后排序值；保存时校验排序值为 0-9999 的整数，且目标父级同层级内排序值唯一。
- 二级展示类目移动到新的一级展示类目下后，系统需重新计算二级类目的最终展示地区 = 新一级类目展示地区 ∩ 当前二级类目展示地区。
- 若移动后父子城市交集为空，后台展示状态更新为“不展示 / 展示地区无交集”，C 端不展示该二级入口。

C 端生效规则：

- 后台保存成功后，C 端下一次请求立即按最新展示类目树生效。
- 保存成功前，C 端仍使用保存前结构；保存失败时，本次移动整体失败，C 端继续展示保存前结构。
- 移动保存成功后，C 端不保留原层级入口，按移动后的层级展示：一级展示类目作为顶部 Tab，二级展示类目作为对应一级 Tab 下的入口。
- 移动保存时需同步完成类目树、层级、父子关系、排序值、城市生效范围、映射后台类目可用性和展示状态重算，避免 C 端读取半更新状态。

关联影响：

- 移动后展示类目完整路径、查询结果路径、导出路径和操作日志对象路径需按最新树结构展示。
- 映射后台类目关系不因移动丢失，仍按已选后台类目 ID 保留；若映射后台类目状态或路径变化，展示状态和映射路径文案同步刷新。
- 原父级若移动后不再有子类目，列表需刷新展开箭头和删除限制判断。
- 移动后的类目进入新父级的批量排序范围，并从原父级的批量排序范围移除。
- 历史曝光、点击、订单等统计不回刷，仍按历史发生时的类目 ID 归集；后台当前列表和 C 端请求按最新树结构展示。

商品排序联动：

- 展示类目移动时不生成新的展示类目 ID；该类目已保存的商品排序继续按「展示类目 ID + 商品 ID」保留，不因父级、层级或完整路径变化而重置。
- 移动本身不修改当前展示类目的后台类目映射，因此当前类目的商品池和已配置商品排序保持不变；C 端仅按移动后的类目层级和路径展示原商品池及原排序结果。
- 当前类目的商品排序与新父级的商品排序仍为两个独立作用域，不合并、不继承，也不使用新父级的商品排序覆盖当前类目配置。
- 二级展示类目移动到其他一级展示类目下，或提升为一级展示类目时，原商品排序保持不变；无子类目的一级展示类目降为二级展示类目时，原商品排序同样保持不变。
- 若移动完成后运营另行修改当前展示类目的后台类目映射，系统再按最新映射重算商品池：新进入商品的排序值初始化为 100；因映射调整移出商品池的商品不再展示，同时删除该商品在当前展示类目下的历史排序配置，后续重新进入商品池时按默认排序值 100 重新初始化。

保存后反馈：

- 移动保存成功后，系统同步更新当前类目的层级、上级类目、排序值、地区覆盖、展示状态和更新时间。
- 保存成功后列表按最新树结构和排序值重新展示，跳转到移动后类目所在页并高亮当前类目。
- 移动操作写入操作日志，记录原上级、目标上级、原层级、目标层级、展示地区影响、商品排序保留规则和 C 端生效说明。

#### 3.3.10 展示类目商品排序

入口与定位：

- 展示类目行提供「商品排序」入口，用于配置该展示类目下商品在 C 端的展示顺序。
- 商品排序属于展示类目维度的运营配置，不修改后台类目结构，不修改商品所属后台类目，也不影响商家创建商品时选择后台类目。
- 一级、二级展示类目均可配置商品排序；实际商品池均由当前展示类目映射的后台类目动态计算。
- 普通类目新增、编辑、移动和批量排序只处理类目树结构；商品排序通过独立入口处理，避免类目结构配置和商品运营配置混在同一流程。

排序作用域：

- 商品排序按「展示类目 ID + 商品 ID」保存。
- 商品排序无城市维度；展示地区仍由展示类目自身控制。
- 同一商品在不同展示类目下可配置不同排序。

商品池计算：

- 商品池来源为当前展示类目映射后台类目下符合条件的商品集合。
- 当展示类目映射多个后台类目时，商品池合并后按商品 ID 去重。
- 商品同时命中多个映射后台类目时，列表展示其命中的后台类目路径；排序配置仍只保存一条商品 ID 维度配置。
- 商品排序列表仅展示上架状态商品；下架商品不进入商品排序配置列表。
- 后台类目取消映射时，对应商品移出当前展示类目商品池，并删除该商品在当前展示类目下的历史排序配置；后续重新映射进入时按默认排序值 100 初始化。后台类目临时禁用时，对应商品不在 C 端展示，但不视为取消映射。
- 商品从后台父类目迁移到其后台子孙类目后，若仍属于同一展示类目（前台类目）商品池，则保留该商品在当前展示类目下的原商品排序值；只有实际退出该展示类目商品池时才删除历史排序配置。新进入商品池的商品排序值初始化为 100。

页面字段与查询：

- 商品排序页展示当前展示类目路径。
- 列表字段包括：商品 ID、商品名称、后台类目路径、商户 ID、商户名称、售卖价格、商品排序值。
- 筛选区仅保留商品搜索，支持按商品 ID 或商品名称查询；点击「查询」后刷新商品列表。
- 当前展示类目路径以紧凑只读信息条展示；商品搜索与查询位于左侧查询组，Excel 导出、导入位于右侧文件操作组，通过间距和分隔线明确区分。导入、导出均使用次级按钮，弹窗底部「保存排序」为当前流程唯一主按钮。
- 查询条件只影响当前页面展示，不改变已保存的排序配置。
- 商品排序页提供「导出 Excel」和「导入 Excel」入口，不提供勾选商品或页面批量设置统一排序值功能。

排序规则：

- 商品排序值越小越靠前，与类目排序值方向保持一致。
- 商品排序值允许相同；排序值相同时由后端默认排序逻辑兜底。
- 所有商品初始化排序值为 100，页面和导出文件均直接展示 100，不允许留空。
- 商品排序值必填，支持 0-9999 的整数；小于 100 表示相对默认值前移，大于 100 表示相对默认值后移。
- 商品排序值表头右侧展示问号说明入口；鼠标悬浮时完整展示“数值越小排序越靠前；默认排序值为 100；允许相同排序值，相同时由后端默认排序逻辑兜底；支持 0-9999 的整数。”
- 排序保存后，C 端下一次请求按最新展示类目商品排序生效。

Excel 导出：

- 导出范围为当前展示类目商品池内的全部上架商品，不受页面商品搜索条件影响，确保业务获得完整可编辑清单。
- 导出文件字段顺序：商品 ID、商品名称、后台类目路径、商户 ID、商户名称、售卖价格、商品排序值。
- 商品 ID 是导入匹配键；商品名称、类目、商户和价格仅用于识别，业务只允许修改商品排序值。
- 导出文件采用 Excel 格式，文件名包含展示类目路径和“商品排序”。

Excel 导入：

- 业务上传由当前页面导出的 Excel 文件；单个文件不超过 5 MB。
- 系统校验必需列、商品 ID 是否为空、商品 ID 是否重复、商品是否存在且仍属于当前展示类目、商品是否为上架状态，以及商品排序值是否为 0-9999 的必填整数。
- 上传后在当前商品排序弹窗内展开校验结果，展示文件名、可导入条数、失败条数，以及每条商品的原排序值、新排序值和错误原因。
- 只有全部数据校验通过时才允许确认导入；确认后整批生效，任一条失败时不得部分更新。
- 导入只更新文件中包含的商品排序值；未包含的商品保持原排序值不变。
- 支持取消导入和重新选择文件；导入成功后刷新当前商品列表及排序。

保存与日志：

- 单条保存时校验商品排序值格式；为空或非法值需在当前输入框定位并提示“商品排序值必填，需为 0-9999 的整数”。
- 点击保存时先完成当前列表全部商品排序值校验；任一商品排序值非法时整次保存不生效，不允许前半部分商品排序值被提前更新。
- 保存成功后提示“商品排序已保存，C 端下一次请求按最新排序值生效”。
- 商品排序操作写入展示类目操作日志，操作类型为“排序”，记录展示类目、变更商品数和操作人。
- Excel 导入、导出均写入展示类目操作日志；导入记录更新商品数，导出记录导出商品数。
- 删除展示类目时，关联商品排序配置随展示类目配置删除；历史日志和统计数据保留展示类目 ID 与当时配置。

#### 3.3.11 删除规则

删除限制：

- 类目下存在子类目时，不允许删除。
- 类目已关联商品或被其他配置引用时，不允许删除。
- 展示类目存在曝光数据或操作日志时，仍可按删除规则删除当前配置，但历史日志和统计数据保留类目 ID 与当时展示地区。
- 删除前需二次确认。

拦截与成功反馈：

- 若类目存在子类目，系统提示“该类目下存在子类目，不允许删除。请先处理子类目。”，不执行删除。
- 若类目允许删除，确认后从列表移除，刷新分页统计和父级展开状态，并提示删除成功。

#### 3.3.12 批量排序

排序范围：

- 支持运营按同一父级类目下的子类目批量调整排序。
- 一级类目之间排序、同一一级下的二级类目排序分别独立处理。
- 一级类目排序影响对应城市 C 端顶部 Tab 顺序。
- 二级类目排序影响对应城市 C 端一级 Tab 下方图片入口顺序。
- 排序值在全局类目结构下维护，不按城市单独配置排序；某城市过滤掉部分类目后，剩余类目按原排序值相对顺序展示。
- 点击「批量排序」打开排序弹窗，系统根据当前展示类目列表实时生成可排序范围，选择排序范围后展示该父级下真实可排序类目列表。

弹窗交互：

- 排序弹窗打开时，列表带出当前类目的原始排序值。
- 排序弹窗支持拖拽调整顺序；拖动完成后弹窗内序号和排序值自动按新顺序刷新，排序值按 1 为间隔重新生成。
- 排序弹窗底部规则分两行展示：第一行说明“排序只支持同一父级下调整；弹窗打开时带出当前列表中的原始排序值。”，第二行说明“拖拽调整后，系统按 1 为间隔重新生成排序值；保存后刷新列表顺序。”
- 不支持跨层级、跨父级排序。

保存后反馈：

- 保存后更新排序值、更新时间和列表行顺序，C 端展示按拖拽后的新顺序生效。

#### 3.3.13 启用/禁用规则

展示影响：

- 禁用类目后，该类目在所有城市均不在 C 端展示。
- 禁用一级类目时，其下二级类目在所有城市也不在 C 端展示。
- 禁用二级类目时，该二级入口在所有城市不在 C 端展示。

确认提示：

- 禁用一级类目时，确认弹窗提示“禁用一级类目后，其下二级类目不会在 C 端展示。”
- 禁用二级类目时，确认弹窗提示“禁用二级类目后，该二级入口不会在 C 端展示。”
- 禁用父级类目时，不修改子级类目的状态字段；父级禁用期间，其下所有子级类目在C 端展示和业务生效判断中均不生效。父级重新启用后，子级类目按自身启用/禁用状态恢复生效。

状态展示：

- 列表中通过“配置状态”和“展示状态”区分自身配置与实际展示结果；二级类目自身为启用但受一级类目禁用影响时，配置状态仍为“启用”，展示状态为“不展示 / 上级已禁用”。

操作反馈：

- 点击启用或禁用时需要二次确认。
- 确认后更新列表中的配置状态、展示状态、更新时间和对应启用/禁用操作文案，并提示状态更新成功。

#### 3.3.14 导出规则

导出范围：

- 展示类目管理支持按当前筛选条件导出。
- 单次导出上限 1000 条；超过 1000 条时，系统提示缩小筛选范围后再导出。

导出内容：

- 导出格式按层级拆列：一级类目、二级类目。
- 导出字段顺序：一级类目、二级类目、类目 ID、类目层级、上级类目、映射后台类目、展示地区、城市 ID、城市名称、类目图片、排序值、配置状态、展示状态、更新时间。
- 类目图片字段导出运营配置图片的图片标识或图片地址，以实际存储口径为准；未配置时为空。商品图兜底属于运行时计算结果，不作为类目图片配置导出，避免再次导入时被误识别为人工配置。

导出交互：

- 点击「导出」后需要二次确认，确认后生成导出文件。
- 导出日志记录操作对象为“展示类目”，变更内容需说明按当前筛选条件导出展示类目。

#### 3.3.15 操作日志

入口与页面：

- 展示类目管理支持查看操作日志，入口位于类目列表标题栏右侧，与导出、批量排序和新增等页面级动作同区展示，逻辑与后台类目操作日志一致。
- 点击「操作日志」后进入独立的「展示类目操作日志」页面，不使用弹窗承接。
- 操作日志页支持点击「返回展示类目管理」回到展示类目列表页。

日志范围与字段：

- 日志记录范围：新增、编辑、移动、删除、启用、禁用、批量排序、商品排序、商品排序 Excel 导入、导出。
- 日志字段：操作时间、操作类型、操作人、操作对象、变更内容、操作结果。
- 操作对象展示类目名称和类目 ID；批量排序按排序范围记录，如“一级类目”“数码下的二级类目”；商品排序按展示类目记录。
- 变更内容需说明本次操作的关键变化，例如新增位置、映射后台类目变化、展示地区变化、图片变化、状态变化、类目排序范围和商品排序变更商品数。
- 日志默认按操作时间倒序展示。

筛选与分页：

- 操作对象、操作类型、操作人和查询、重置按钮在同一筛选操作区内对齐展示；日志标题栏固定在该区域上方，不得覆盖任何筛选项。
- 支持按操作对象模糊查询，匹配展示类目名称或类目 ID。
- 支持按操作类型筛选：全部、新增、编辑、移动、删除、启用、禁用、排序、导入、导出。
- 支持按操作人筛选：全部或具体操作人。
- 日志列表支持分页，默认 50 条/页，支持切换每页 50 / 100 / 200 条。
- 查询结果为空时展示空状态，不影响原展示类目列表。

### 3.4 原型说明

原型覆盖以下页面和状态：

- 左侧独立菜单顺序：展示类目管理、后台类目管理
- 后台类目管理列表页
- 展示类目管理列表页
- 展示类目只支持一级、二级
- 展示类目列表展示映射后台类目
- 展示类目新增 / 编辑弹窗支持选择映射后台类目和展示地区
- 新增/编辑类目弹窗
- 二级展示类目图片上传字段
- 图片未上传时点击「+」打开本地图片选择
- 图片已上传时，配置页图片预览可点击放大
- 图片已上传后支持「更新图片」重新选择图片，支持「删除图片」二次确认后清空
- 类目图片列表字段仅展示运营配置图；未配置时展示「-」，商品图兜底仅在 C 端运行时生效，不回显配置后台
- C 端二级类目头图按“运营配置图 > 排序第一有效商品主图”取图；删除配置图、商品排序变化、商品失效或后台类目映射变化后自动重算
- 列表页类目图片缩略图可点击放大，放大查看层支持关闭按钮、点击遮罩和 Esc 关闭
- 一级展示类目不展示图片字段
- 新增子类时上级类目自动带入并置灰
- 编辑时上级类目和类目层级置灰
- 类目行支持「移动」入口，移动弹窗支持搜索并选择目标上级、自动推导移动后层级、填写移动后排序值
- 移动类目时展示受影响子类目数量、移动后最深层级和 C 端立即生效说明
- 移动保存时校验不可选择自身或子孙类目、不可超过最大层级、目标父级同层级排序值不可重复
- 展示类目移动后 C 端按最新展示类目树立即生效
- 展示类目行支持「商品排序」入口，打开后展示当前展示类目下上架状态商品及商品排序值
- 商品排序弹窗支持按商品 ID / 商品名称查询，并提供商品排序 Excel 导入、导出入口
- 商品排序值默认 100 且必填，数值越小越靠前；单条保存时校验为 0-9999 整数，保存后写入排序日志
- 商品排序值表头右侧展示问号，悬浮时完整说明排序方向、默认值、重复值兜底和取值范围
- 商品排序 Excel 导出完整上架商品池；导入支持文件选择、逐条校验预览、全量校验通过后整批生效，并写入导入、导出日志
- 移动类目的目标上级选择器打开时展示全部合法候选，不受当前上级名称过滤
- 排序值标题右侧问号悬浮说明，说明内容完整展示
- 类目列表默认按树形结构展示，同一父级同一层级按排序值升序排列，更新时间不参与默认排序
- 单个新增保存后跳转到新增类目所在页并高亮；批量新增保存后跳转到本次新增排序最靠前类目所在页并高亮当前页新增行；编辑保存后跳转到该类目调整后所在页并高亮
- 单个新增、批量新增、编辑保存时校验类目名称全局唯一
- 后台类目和展示类目的单个新增、批量新增、编辑保存均校验类目名称去除首尾空格后为 1-20 个字符；单个输入展示实时字符计数，批量输入超限时逐条提示具体行号、完整类目名称和超限原因，且前端与服务端口径一致、不静默截断
- 单个新增、批量新增、编辑保存和移动保存时校验同一父级同一层级排序值唯一
- 批量新增时校验生成出的排序值均在 0-9999 范围内，且不与已有同级类目重复
- 分页页码、上一页、下一页和每页条数可点击切换；每页条数支持 50 / 100 / 200，默认 50 条/页
- 新增 / 批量新增保存后列表即时新增数据，并刷新分页统计
- 编辑保存后当前列表行即时更新
- 删除、启用/禁用、批量排序保存后列表数据即时更新
- 批量排序弹窗的排序范围和原始数据来自当前列表实时数据
- 删除、禁用等风险提示
- 类目展开 / 收起
- 批量排序弹窗及拖拽排序
- 后台类目导出确认
- 展示类目导出确认
- 后台类目操作日志页面、日志筛选、日志列表和分页
- 展示类目操作日志页面、日志筛选、日志列表和分页
- 新增 / 编辑保存校验和成功提示

### 3.5 暂不包含

- 类目和商品的批量绑定。
- 类目和属性、品牌的完整配置。
- 展示类目跳转专题页、活动页、搜索词配置。
- 审批流。
- 城市维度独立排序；当前版本排序值全局维护，按城市过滤后保持相对顺序。
